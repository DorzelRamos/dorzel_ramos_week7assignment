Selectors & Filters

Basic Selectors

JQuery Selectors use a dollar sign ($) at the beginning of each of their selectors. It also uses hashtags (#) for ids, periods (.) for classNames, and can target tag elements similar to CSS. JQuery selectors can also use the asterix, or wild card (*) to target all the elements in the page

Basic Filters

Filters come after a semicolon (:) and can filter things by first and last, odds and evens, gt (greater than), lt (less than), eq (equal to), animated, focus, and not(expr), or non expressions

Advanced Selectors

When the (>) selector is used that means "select this tag only if its an immediate child of the parent". Descendant selectors select anything inside the element and doesn't have to be an immediate child. The adjacent selector (+) means "select this element right after this element". The (~) selector means "select each element as long as it has this element as its sibling", so elements following the previous one will be selected.

Attribute Filters

These lets you perform tests to see if attributes are being used. Attribute filters are used inside [] brackets after the tag. These can filter out classes and ids. The (^=) filter means pick out a name that starts with this string. The (*=) filter means pick out an element that contians this certain string

Advanced Filters

The contains filter selects elements with a certain text. The parent filter means "select these tags as long as their parents". So anything with a child or a textnode is considered a parent. The has filter selects paragraphs with a certain class. The child filter selects paragraph tags inside of an element. This can be used with first, last, and last-type-of. The n-th filter selects the number of the child inside an element. So if a paragraph is the third child of an element it can select the third child
